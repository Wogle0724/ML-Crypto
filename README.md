# Crypto Predictor

A cryptocurrency price prediction platform built with a microservices architecture.
Prices are fetched via Airflow DAGs, stored in MySQL, served through a Node.js REST API,
and predictions are generated by a Python FastAPI ML service backed by a PyTorch LSTM model.
All experiments are tracked in MLflow and surfaced in a React dashboard.

## Services

| Service           | URL                          | Notes                    |
|-------------------|------------------------------|--------------------------|
| Frontend          | http://localhost:3000        | React + Vite dev server  |
| Backend API       | http://localhost:4000        | Node.js + Express        |
| ML Service        | http://localhost:8000        | Python FastAPI           |
| Airflow           | http://localhost:8080        | Login: admin / admin     |
| MLflow            | http://localhost:5000        | Experiment tracking UI   |
| MySQL             | localhost:3306               | DB: crypto_db            |
| Redis             | localhost:6379               | Cache                    |

## Getting Started

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (v24+)
- Docker Compose v2 (`docker compose` — note: no hyphen)

### 1. Configure environment

The `.env` file is pre-populated with development defaults.
Before first run, generate a real Airflow Fernet key:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

Paste the output into `.env` as `AIRFLOW_FERNET_KEY`, and set a random string for `AIRFLOW_SECRET_KEY`.

### 2. Start all services

```bash
docker compose up --build
```

MySQL and Redis will start first. Once healthy, the backend, ml-service, and Airflow init container start.
After `airflow-init` completes, the Airflow webserver and scheduler launch.

> First build takes several minutes due to PyTorch in the ML service and pipeline images.

### 3. Verify everything is running

```bash
# Backend mock price data
curl localhost:4000/api/prices/bitcoin

# ML service dummy prediction
curl -X POST localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"coin": "bitcoin"}'

# Backend forwarding prediction from ml-service
curl localhost:4000/api/predict/bitcoin
```

Then open:
- **http://localhost:3000** — React dashboard (placeholder chart + prediction overlay)
- **http://localhost:8080** — Airflow UI, DAGs: `fetch_prices`, `train_model`
- **http://localhost:5000** — MLflow experiment tracking

### 4. Stop services

```bash
docker compose down
```

To also remove all volumes (database data, MLflow artifacts):

```bash
docker compose down -v
```

## Project Structure

```
crypto-predictor/
├── frontend/        # React + Vite UI
├── backend/         # Node.js + Express REST API
├── ml-service/      # Python FastAPI + PyTorch inference
├── pipeline/        # Apache Airflow DAGs (data fetch + model training)
```
